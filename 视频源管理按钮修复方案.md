# 视频源管理按钮修复方案

## 问题概述

视频源管理页面的 6 个操作按钮中，有 4 个存在功能缺陷：

| 按钮 | 当前状态 | 问题描述 |
|------|----------|----------|
| 检查 | 仅 console.log | 无实际检查功能 |
| 增量 | 调用 collectLatest() | 忽略 sourceId，采集全部源而非当前源 |
| 全量 | 调用 collectAll() | 忽略 sourceId，采集全部源而非当前源 |
| 编辑 | 设置 editingSource 状态 | 无弹窗 UI，点击无反应 |
| 禁用/启用 | ✅ 正常工作 | - |
| 删除 | ✅ 正常工作 | 使用原生 confirm，体验一般 |

---

## 修复方案（对齐原网站设计）

### 1. 检查按钮

**目标**: 实现视频源可用性检查，更新健康状态

**原网站设计要点**:
- 调用 `CMSAdapter.getTypes()` 接口（而非 getList）
- 更新字段：`healthStatus`, `lastCheckAt`, `lastSuccessAt`, `failCount`, `avgResponseTime`
- 计算响应时间，与历史取平均
- 返回布尔值 `healthy`

**修改文件**:
- `packages/core/src/services/collectorService.ts` — 添加 `checkSource` 方法
- `packages/core/src/store/createStore.ts` — 添加 `checkVideoSource` 方法
- `apps/desktop/src/pages/SourceManagerPage.tsx` — 调用检查逻辑，显示 toast

**实现逻辑**:
```typescript
// collectorService.ts
async checkSource(sourceId: string): Promise<{ healthy: boolean; responseTime: number }> {
  const source = await this.db.getVideoSourceById(sourceId);
  if (!source) return { healthy: false, responseTime: 0 };
  
  const adapter = new CMSAdapter(source.baseUrl, source.rateLimit);
  const startTime = Date.now();
  
  try {
    await adapter.getTypes();
    const responseTime = Date.now() - startTime;
    
    const avgResponseTime = source.avgResponseTime 
      ? Math.round((source.avgResponseTime + responseTime) / 2)
      : responseTime;
    
    await this.db.updateSourceHealth(sourceId, {
      healthStatus: 'HEALTHY',
      lastCheckAt: new Date().toISOString(),
      lastSuccessAt: new Date().toISOString(),
      failCount: 0,
      avgResponseTime
    });
    
    return { healthy: true, responseTime };
  } catch {
    await this.db.updateSourceHealth(sourceId, {
      healthStatus: 'DOWN',
      lastCheckAt: new Date().toISOString(),
      failCount: (source.failCount || 0) + 1
    });
    return { healthy: false, responseTime: Date.now() - startTime };
  }
}
```

### 2. 增量按钮

**目标**: 仅采集当前点击的视频源的最新数据（第 1 页）

**原网站设计要点**:
- 使用 `source.code` 作为参数（而非 id）
- 添加幂等性检查（防止重复点击同一类型任务）
- 返回任务标识，显示 toast 提示

**修改文件**:
- `packages/core/src/services/collectorService.ts` — 添加 `collectSourceLatest` 方法
- `packages/core/src/store/createStore.ts` — 添加 `collectSourceLatest` 方法，管理任务状态
- `apps/desktop/src/pages/SourceManagerPage.tsx` — 修复调用参数，添加 toast 提示

**实现逻辑**:
```typescript
// collectorService.ts
async collectSourceLatest(sourceCode: string): Promise<{ taskId: string; collected: number }> {
  const source = await this.db.getVideoSourceByCode(sourceCode);
  if (!source || !source.isEnabled) return { taskId: '', collected: 0 };
  
  const taskId = `${sourceCode}-INCREMENTAL-${Date.now()}`;
  const { media } = await this.collectFromSource(source.id, source.baseUrl, source.rateLimit, 1);
  
  return { taskId, collected: media.length };
}
```

### 3. 全量按钮

**目标**: 仅采集当前点击的视频源的全量数据

**原网站设计要点**:
- 使用 `source.code` 作为参数
- 添加幂等性检查
- 返回任务标识，显示 toast 提示

**修改文件**:
- `packages/core/src/services/collectorService.ts` — 添加 `collectSourceAll` 方法
- `packages/core/src/store/createStore.ts` — 添加 `collectSourceAll` 方法，管理任务状态
- `apps/desktop/src/pages/SourceManagerPage.tsx` — 修复调用参数，添加 toast 提示

**实现逻辑**:
```typescript
// collectorService.ts
async collectSourceAll(sourceCode: string): Promise<{ taskId: string; collected: number; pages: number }> {
  const source = await this.db.getVideoSourceByCode(sourceCode);
  if (!source || !source.isEnabled) return { taskId: '', collected: 0, pages: 0 };
  
  const taskId = `${sourceCode}-FULL-${Date.now()}`;
  let collected = 0;
  let pages = 0;
  let page = 1;
  const config = await (new SystemConfigService(this.db)).getCollectConfig();
  
  while (page <= config.maxPages) {
    const { media, pagecount } = await this.collectFromSource(source.id, source.baseUrl, source.rateLimit, page);
    collected += media.length;
    pages++;
    if (page >= pagecount) break;
    page++;
  }
  
  return { taskId, collected, pages };
}
```

### 4. 编辑按钮

**目标**: 实现卡片内联编辑模式

**原网站设计要点**:
- 卡片内联编辑（非弹窗）
- 仅允许编辑 name 和 baseUrl
- code 显示为只读输入框（灰色背景）
- 添加取消和保存按钮
- priority 通过拖拽管理，rateLimit 通过速率按钮管理，isEnabled 通过启用/禁用按钮管理

**修改文件**:
- `apps/desktop/src/pages/SourceManagerPage.tsx` — 实现内联编辑模式

**实现逻辑**:
1. 点击编辑按钮进入编辑模式（卡片变为表单视图）
2. 编辑表单字段：
   - code: 只读输入框，灰色背景
   - name: 必填文本输入框
   - baseUrl: 必填文本输入框
3. 双列布局：code 和 name 第一行，baseUrl 独占第二行
4. 保存时校验并调用 `upsertVideoSource` 更新数据库
5. 取消时恢复卡片正常视图

## 注意事项

- 编辑功能仅修改 name 和 baseUrl，其他字段通过专用按钮管理
- 检查功能调用后需刷新列表以更新健康状态显示
- 增量/全量采集时需添加幂等性检查防止重复点击
- 桌面端数据库 migration 需要添加新字段
